# 목표

기술 면접에서 자료구조에 대한 질문을 받았을 때, 자신있게 대답 하지 못하는 걸 보고 공부를 시작했습니다. <br/>
이걸 계기로 자료구조에 대해 자료구조에 대한 기본적인 개념과 종류, 각각의 특징을 알아보고자 합니다. <br/>

# 자료구조(Data Structure)

## 자료구조란?

> 효율적인 접근 및 수정을 가능케 하는 자료의 조직, 관리, 저장등을 의미합니다.

데이터의 특성과 크기, 주요 사용법에 따라 여러가지 종류의 자료구조 중 하나를 선택할 수 있습니다.

## 자료구조의 종류

- 단순 자료구조(Primitive Data Structure)
  - int
  - float
  - char
  - ...
- 복합 자료구조(Composite Data Structure)
  - 선형 자료구조
    - 배열(Array)
    - 연결 리스트(Linked List)
    - 스택(Stack)
  - 비선형 자료구조
    - 해시 테이블(Hash Table)
    - 트리(Tree)
    - 그래프(Graph)
    - 트라이(Trie)
    - 집합(Set)
    - 맵(Map)

## 단순 자료구조(Primitive Data Structure)
단순 자료구조는 프로그래밍 언어에서 기본적으로 제공하는 자료구조입니다.
정수, 실수, 문자, 문자열 등이 있으며, 이러한 자료구조는 메모리에 저장되어 연산이 가능합니다.

- int : 정수형 데이터를 저장하는 자료구조
- float : 실수형 데이터를 저장하는 자료구조
- char : 문자형 데이터를 저장하는 자료구조
- bool : 논리형 데이터를 저장하는 자료구조

## 복합 자료구조(Composite Data Structure)

### 1. 선형 자료구조

선형적으로 배열되어 있는 자료구조입니다. 데이터간의 관계가 1:1로 연결되어 있습니다.

- 배열(Array)
- 연결 리스트(Linked List)
- 스택(Stack)
- 큐(Queue)

### 2. 비선형 자료구조

비선형적으로 연결하며 데이터간의 관계가 1:N 또는 N:N 등으로 복잡하게 연결되는 계층 구조를 갖습니다.

- 해시 테이블(Hash Table)
- 트리(Tree)
- 그래프(Graph)
- 트라이(Trie)
- 집합(Set)
- 맵(Map)

## 선형 자료구조

### 1) 배열(Array)

> 배열은 모든 요소가 순차적으로 배열된 구조로, **연속된 메모리 위치**에 저장된 **동일한 데이터 유형**의 요소를 가집니다.
배열에 저장된 데이터는 인덱스를 통해 접근할 수 있습니다. <br/>

![](https://velog.velcdn.com/images/r_eum/post/945ffce2-9d21-40ed-9dea-9a1ae6d0a48c/image.png)

배열은 순서를 보장해야 하는 형태의 데이터를 저장할 때 사용하면 좋은 자료구조입니다.

배열은 선언될 때 고정된 메모리를 할당 받기 때문에 크기를 변경할 수 없습니다.

이러한 선언은 정적 또는 컴파일 타임 메모리 할당이라고 합니다. 즉 배열 요소의 메모리는 컴파일 시간에 할당되며, 실행 중에 크기를 변경할 수 없습니다.


<br/>

### 2) 연결 리스트(Linked List)

연결 리스트는 데이터 요소들을 노드라는 객체로 구성하며, 노드는 데이터와 다음 노드에 대한 참조로 구성됩니다.
인덱스가 아닌 참조를 통해 순차적으로 연결되는 구조입니다. <br/>
첫 번째 요소를 참조하는 **head**, 마지막 요소를 참조하는 **tail**이 있습니다. <br/>
![](https://velog.velcdn.com/images/r_eum/post/9bc5df9f-ef4d-4b88-b99c-124f92355d67/image.png)
리스트는 데이터의 추가 및 삭제시 구조를 재구성할 필요가 없이 참조만 변경하면 되기 때문에 동적인
메모리 할당에 용이합니다. 하지만 배열보다 메모리 사용량이 더 많고, 데이터를 찾기위해 순차적으로 탐색해야 하는 단점이 있습니다.

또한, 연결 리스트에는 3가지의 유형이 존재합니다
1. 단일 연결 리스트
2. 이중 연결 리스트
3. 원형 연결 리스트

**단일 연결 리스트** <br/>
![](https://velog.velcdn.com/images/yukina1418/post/23a8346a-0c9e-4245-913f-5f3cb3c3174a/image.png)

그림과 같이 Next 포인터만 가지고 있습니다.

**이중 연결 리스트** <br/>
![](https://velog.velcdn.com/images/yukina1418/post/e00d8b46-4069-4bed-92f8-acef2fb0bc5b/image.png)

그림과 같이 Next, Prev 포인터를 가지고 있습니다. head와 tail의 각각 첫번째 포인터와 마지막 포인터는 Null을 가리키고 있습니다.

**원형 연결 리스트** <br/>
![](https://velog.velcdn.com/images/yukina1418/post/fa80f60a-11cd-4a8e-9b93-0fe705260a0a/image.png)

이중 연결 리스트와 비슷하지만, 마지막 노드가 첫번째 노드를 가리키고 있습니다.

<br/>

### 3) 스택(Stack)

LIFO(Last In First Out) 메커니즘을 가지는 선형 데이터 구조입니다.
보통 배열로 구현을 많이 하며, 브라우저 히스토리가 대표적인 예시라고 생각됩니다. <br/>

스택은 데이터를 넣는 작업을 푸시(push), 데이터를 빼는 작업을 팝(pop)이라고 합니다. <br/>

![](https://media.geeksforgeeks.org/wp-content/cdn-uploads/20221219100314/stack.drawio2.png)


### 4) 큐(Queue)

FIFO(First In First Out) 메커니즘을 가지는 선형 데이터 구조입니다.

큐는 먼저 입력된 요소가 먼저 처리되는 구조로, 요소를 추가하는 작업을 인큐(enqueue), 요소를 제거하는 작업을 디큐(dequeue)라고 합니다.

큐는 배열이나 리스트로 구현될 수 있으며, 배열로 구현된 큐는 원형 큐로 구현될 수 있습니다. <br/>

이벤트 루프의 태스크 큐가 대표적인 예시라고 생각됩니다.
